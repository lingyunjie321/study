# 检索召回策略优化说明

## 背景

平台文档搜索（platform documentation search）在数据分析和 ETL 场景中是非常核心的能力。用户通常用自然语言查询（如 "CREATE TABLE syntax"、"COPY INTO options"），期望系统返回精准且多样化的结果。然而单纯依靠向量相似度搜索存在以下局限：

- **语义漂移**：向量模型对短文本_QUERY_（尤其是 SQL 关键字）的嵌入质量不稳定
- **同义词覆盖弱**：用户搜 "table" 可能搜不到 "relation" 相关内容
- **结果扎堆**：top_k 结果全部来自同一文档的不同 chunk，缺乏多样性
- **字段权重缺失**：title、hierarchy、keywords 等结构化字段的匹配没有获得应有的权重

## 优化策略

### 1. BM25 全文匹配（混合召回）

在向量搜索之外引入经典的 BM25 排序算法，对 chunk_text 进行词项匹配打分：

```python
bm25_k1: float = 1.5   # 词频饱和参数
bm25_b: float = 0.75   # 文档长度归一化参数
```

BM25 得分与向量得分按权重混合：

| 权重配置 | 默认值 | 说明 |
|---------|-------|------|
| `vector_weight` | 0.50 | 向量语义相似度 |
| `text_weight` | 0.30 | BM25 词项匹配 |
| `title_weight` | 0.10 | 标题命中加分 |
| `hierarchy_weight` | 0.05 | 层级路径命中加分 |
| `keywords_weight` | 0.05 | 关键词命中加分 |

### 2. 查询扩展（Query Expansion）

对于短查询（≤2 个词），自动补充扩展词：

| 原始词 | 扩展词 |
|-------|--------|
| create | define, add, new |
| table | relation, dataset |
| select | query, fetch, read |
| insert | add, load, write |
| join | combine, merge, union |
| index | performance, speed, optimize |

多个扩展查询并发搜索，结果按 `vector_score` 去重合并。

### 3. 字段级加权提升（Multi-field Boosting）

```python
# 标题加分规则
if query_lower in title_lower:      title_boost = 0.8  # 完整包含
elif any(term in title_lower ...):  title_boost = 0.4  # 部分词命中

# 层级路径加分规则（同上）
# hierarchy_boost: 完整包含 0.6，部分命中 0.3

# 关键词命中率加分
keywords_boost = kw_match_count / total_keywords * 0.5
```

### 4. 多样性约束（Diversity Boosting）

防止 top_k 结果全部来自同一文档的不同 chunk：

```python
max_chunks_per_doc: int = 3    # 同一文档最多返回 3 个 chunk
diversity_decay: float = 0.15   # 超过后每次额外降权 15%
```

超出限制的 chunk 其 `final_score` 会乘以 `(1 - decay * excess)` 惩罚系数。

### 5. 结果对比评估（Baseline Comparison）

`--compare` 模式同时执行 baseline 和优化版搜索，输出：

- 两个版本的召回数量和重叠率
- 新发现的结果（baseline 中未出现）
- 每个结果的分数拆解（vector / text / title / hierarchy / keywords）

## 文件结构

```
scripts/
└── optimize_recall/
    ├── __init__.py              # 包导出
    └── evaluate_recall.py      # 主脚本（含所有策略实现）
```

## 使用方法

```bash
# 基础搜索
uv run python -m scripts.optimize_recall.evaluate_recall \
    -p duckdb -q "CREATE TABLE syntax"

# 指定版本和返回数量
uv run python -m scripts.optimize_recall.evaluate_recall \
    -p snowflake -q "COPY INTO options" -v v2.8 --top 20

# 对比 baseline vs 优化版
uv run python -m scripts.optimize_recall.evaluate_recall \
    -p duckdb -q "INSERT" --compare
```

## 核心类

| 类 | 职责 |
|---|------|
| `OptimizedRecall` | 主入口，管理搜索流程和结果聚合 |
| `RecallConfig` | 可配置的所有权重参数 |
| `SearchResult` | 带分项打分的结果数据结构 |
| `BM25` | 独立实现的 BM25 排序算法 |
| `QueryExpander` | 查询词扩展（同义词注入） |
| `DiversityScorer` | 多样性降权处理器 |
