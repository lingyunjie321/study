# Data_engineer - 怎么跑这个项目

## 环境配置

### 技术栈和依赖

| 组件 | 版本 | 说明 |
|------|------|------|
| Python | 3.12+ | 必须 |
| uv | 最新 | 包管理器 |
| LanceDB | 最新 | 向量数据库 |
| FastAPI | 最新 | API 服务 |
| OpenAI Agents SDK | 最新 | Agent 框架 |
| LiteLLM | 最新 | LLM 统一调用 |

### 环境变量

```bash
# LLM API Keys（根据使用的提供者配置）
export OPENAI_API_KEY=sk-...
export ANTHROPIC_API_KEY=sk-ant-...
export DEEPSEEK_API_KEY=...

# 可选
export DATUS_HOME=~/.datus          # 数据目录
export DATUS_VERBOSE=1              # 详细日志
```

### 配置文件

```yaml
# agent.yml 放在项目根目录或 ~/.datus/
agent:
  target:
    provider: openai
    model: gpt-4o

datasources:
  - name: demo
    type: duckdb
    path: ./demo.duckdb
```

## 运行步骤

### 初始化

```bash
# 1. 安装依赖
uv sync

# 2. 初始化配置
datus-agent init

# 3. 配置 LLM（交互式）
datus config set target.provider openai
datus config set target.model gpt-4o
```

### 启动 CLI

```bash
# 启动交互式 CLI
datus-cli --datasource demo

# 带 verbose 日志
datus-cli --datasource demo --verbose
```

### 启动 API 服务

```bash
# 启动 REST API
datus-api

# 指定端口
datus-api --port 8080

# 后台运行
nohup datus-api > api.log 2>&1 &
```

### 启动 MCP 服务

```bash
# 启动 MCP Server
datus-mcp

# 检查 MCP 状态
datus-mcp status
```

## 常见问题排查

### Q1: 初始化失败 "No module named datus"

**检查清单**：

```bash
# 1. 确认已安装
uv pip show datus-agent

# 2. 重新安装
uv sync
uv pip install -e .

# 3. 检查 Python 版本
python --version  # 需要 3.12+
```

### Q2: 数据库连接失败

**检查清单**：

```bash
# 1. 检查数据源配置
cat agent.yml | grep -A5 datasources

# 2. 测试连接
datus-cli --datasource demo /Check SELECT 1

# 3. 查看连接日志
tail -f ~/.datus/logs/datus.log | grep -i connect
```

### Q3: LLM 调用失败

**检查清单**：

```bash
# 1. 检查 API Key
echo $OPENAI_API_KEY  # 不应为空

# 2. 测试 API
curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"

# 3. 切换备用模型
datus config set target.model gpt-4.1-mini
```

### Q4: 向量检索无结果

**检查清单**：

```bash
# 1. 确认文档已初始化
datus-cli /doc-stats --platform duckdb

# 2. 重新初始化文档
datus-cli /init-docs --platform duckdb --force

# 3. 检查 LanceDB 状态
ls -la ~/.datus/data/<project>/datus_db/
```

### Q5: Session 数据丢失

**检查清单**：

```bash
# 1. 查看 Session 存储
ls ~/.datus/sessions/

# 2. 导出 Session
datus-cli /Export session --id <session_id> --format json

# 3. 清理旧 Session
datus-cli /Clear sessions --older-than 7d
```

## 性能调优

### 调整向量检索参数

```python
# 在 agent.yml 中配置
agent:
  retrieval:
    top_k: 20          # 检索结果数
    similarity_threshold: 0.7  # 相似度阈值

# 创建向量索引（加速检索）
datus-cli /optimize-index --platform duckdb
```

### 调整 LLM 参数

```yaml
# agent.yml
agent:
  llm:
    temperature: 0.0    # 生成确定性
    max_tokens: 2048   # 最大输出
    timeout: 30        # 超时秒数
```
