## 📌 项目简介
Data_engineer 是一个数据工程智能体(Data Engineering Agent),通过领域感知推理、语义搜索和持续学习,将自然语言转换为准确的 SQL 语
句。

核心功能列表

• 🧠 自然语言转 SQL:NL → 意图理解 → 语义检索 → SQL 生成
• 📊 Evoluble Context:构建活的知识库,捕获 schema 元数据、参考 SQL、语义模型、指标和领域知识
• 🔄 Subagent 封装:将成熟的领域封装为 scoped chatbot,提供 API/Web/MCP 多种交付方式
• 📈 语义层 & 指标:通过 MetricFlow 集成定义业务指标 YAML,生成跨数据库方言的 SQL

• 🔌 MCP 协议:既是 MCP Server(暴露工具给 Claude Desktop、Cursor),也是 MCP Client
• 🛠 Skills 系统:支持 agentskills.io ⻛格的打包工具、可配置权限和市场支持
• 🗄 多数据库支持:内置 SQLite/DuckDB + 11 种可插拔数据库适配器

• 🤖 多 LLM 提供者:支持 OpenAI、Claude、Gemini、DeepSeek、Qwen、Kimi 等 10+ 提供者
项目类型

• 开发语言:Python 3.12+
• 技术栈:OpenAI Agents SDK + LiteLLM, LanceDB, FastAPI, FastMCP, Streamlit
• 状态:生产可用,活跃维护

🐛 常⻅问题与调试

Q1: SQL 生成结果为空

症状:输入查询后返回空结果
原因:Schema 链接失败,未找到相关表
解决:

```text
  1 # 检查表是否在上下文中
  2 agent.context.schema_info.get('available_tables')
  3 # 手动触发 Schema 检索
  4 agent.run_node("schema_linking", force=True)
```

Q2: LLM 调用超时
症状:请求超时或响应慢
原因:模型负载高或网络问题

解决:

```text
  1 # 切换更快的模型
  2 datus config set target.model gpt-4.1-mini
```

Q3: 向量检索无结果

症状:文档搜索返回空
原因:平台文档未初始化
解决:

```text
  1 from datus.storage.document.doc_init import init_platform_docs
  2 init_platform_docs(platform="duckdb", version="latest")
```

Q4: 数据库连接失败
症状:无法连接到数据源

原因:连接配置错误
解决:

```text
  1 # 检查配置
  2 datus config show
  3 # 验证连接
  4 datus-cli --datasource demo /Check SELECT 1
```

Q5: MCP 工具不可⻅

---

症状:MCP Client 连接后看不到工具

原因:MCP Server 未正确配置
解决:

```text
  1 # 检查 MCP 配置cat .datus/config.yml | grep mcp
  2 # 重启 MCP 服务
  3 datus-mcp restart
```
