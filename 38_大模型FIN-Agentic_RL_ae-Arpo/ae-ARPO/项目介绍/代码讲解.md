## 1. 训练入口 — main_ppo.py

**文件**: `AEARPO/verl_arpo_entropy/verl/trainer/main_ppo.py`

整个训练的入口。用 Hydra 加载 YAML 配置 + 命令行 override，初始化 Ray 集群，启动训练循环。

```python
@hydra.main(config_path=config_path, config_name=config_name)
def main(config):
    # Hydra 自动将 YAML 配置 + 命令行 override 合并为 config 对象
    ray.init(runtime_env={"env_vars": {"TOKENIZERS_PARALLELISM": "true"}})
    # 通过 Ray Actor 将训练循环部署到 GPU 节点
    runner = TaskRunner.remote()
    ray.get(runner.run.remote(config))
```

**设计要点**: 配置在训练代码外部，金融数据和新数据集只需改命令行参数。
**面试考点**: Hydra 的 override 机制如何通过命令行参数切换数据集。

## 3. Rollout — vLLM 推理引擎

**文件**: `AEARPO/verl_arpo_entropy/verl/workers/rollout/vllm_rollout/`

批量生成模型回复，支持工具调用插入。

```python
class vLLMRollout:
    def generate(self, prompts, tools):
        # 1. vLLM 批量生成
        outputs = self.llm.generate(prompts, sampling_params)
        for output in outputs:
            # 2. 检测工具调用标记
            if "<tool_call>" in output.text:
                tool_name, args = parse_tool_call(output.text)
                tool_result = tools[tool_name].execute(args)
                # 3. 工具结果插入，继续生成
                output.text += f"\n<tool_result>{tool_result}</tool_result>\n"
        return outputs
```

**设计要点**: 生成和工具调用共享一个循环，熵监测在生成时实时计算。
**面试考点**: vLLM 训练时和推理时的参数设置有什么区别？

## 4. 工具 Agent — ToolAgent

**文件**: `AEARPO/verl_arpo_entropy/verl/workers/agent/tool_agent.py`

管理工具注册、参数解析和调用。

```python
class ToolAgent:
    def __init__(self, tool_config):
        self.tools = {}  # 从 YAML config 动态注册工具
        for name, cfg in tool_config["tool_instances"].items():
            tool_cls = import_class(cfg["class_path"])
            self.tools[name] = tool_cls(**cfg["params"])

    def execute(self, tool_name, params):
        tool = self.tools[tool_name]
        return tool.run(params)  # search → API 调用, python → subprocess
```

**设计要点**: 工厂模式——工具在 YAML 中配置后动态注册。
**面试考点**: 如何保证 Python 工具执行的安全性？

## 5. Reward 计算 — deep_research.py

**文件**: `AEARPO/verl_arpo_entropy/verl/utils/reward_score/deep_research.py`

对比模型输出和 ground_truth 计算 reward。

```python
def compute_score(solution_str, ground_truth, **kwargs):
    # 从模型输出中提取最终答案
    pred = extract_final_answer(solution_str)
    # 数值比对：相对误差 < 1% 视为正确
    if is_numeric(ground_truth) and is_numeric(pred):
        return 1.0 if abs(pred - ground_truth) / abs(ground_truth) < 0.01 else 0.0
    # 精确匹配
    return 1.0 if str(pred).strip() == str(ground_truth).strip() else 0.0
```

**设计要点**: 1% 容忍度适配金融计算题（四舍五入差异）。
**面试考点**: 为什么用 rule-based reward 而不是训练 reward model？

## 6. Policy Update — dp_actor.py

**文件**: `AEARPO/verl_arpo_entropy/verl/workers/actor/dp_actor.py`

计算 PPO loss，执行梯度更新。

```python
def update_policy(self, trajectories, advantages):
    for mini_batch in split(trajectories, mini_batch_size):
        new_log_prob = self.model(mini_batch).log_prob
        ratio = (new_log_prob - old_log_prob).exp()

        # PPO clipped loss
        loss = -torch.min(
            ratio * advantages,
            ratio.clamp(1 - clip_ratio, 1 + clip_ratio) * advantages
        ).mean()
        loss.backward()

    # FSDP 同步梯度，更新参数
    self.optimizer.step()
```

**设计要点**: Dynamic batch size——按 token 数而不是样本数做 mini-batch，显存利用率更均匀。
**面试考点**: PPO clip 的作用是什么？

class DataLoader:
    def __init__(self, dataset_name, data_path):
        self.data_path = os.path.join(data_path, dataset_name, 'test.jsonl')

    def load_data(self):
        questions, answers = [], []
        with open(self.data_path, "r") as f:

                data = json.loads(line)
                questions.append(data["question"])
                answers.append(data["answer"])

## 8. 金融数据生成扩展 — finance_economics.py

**文件**: `AEARPO/verl_arpo_entropy/examples/data_preprocess/finance_economics.py`


```python
def curated():    # 精选核心题 (144题)
    add("A company reports revenue of $12.5M...", "Gross Profit =...", D)

def templated():  # 模板化数值变体 (~4080题)
    for i in range(85):
        rev = rand_range(15000)  # 随机化参数 ±20%
        add(f"Revenue ${rev}...", f"Calculated net income: {ni}", D)

def conceptual(): # 概念分析题 (15题)
    add("What is the difference between...", "Detailed comparison...", D)
```

**设计要点**: 三层架构——核心题保证教育价值，模板题保证数据量，概念题保证多样性。参数 ±20% 随机化避免答案记忆。
**面试考点**: 为什么不全用 HuggingFace 上的公开金融数据集？

**参考答案**：五个原因。(1) 任务形式不匹配：FinQA/ConvFinQA 等公开数据集是阅读理解任务（给财报原文找答案），而 AEARPO 需要独立的多步数值计算推理题，两种范式不能直接复用。(2) 公开数据集缺少 chain-of-thought：reward 计算需要完整分步推导过程做 rule-based 匹配，公开数据集通常只有最终答案。(3) 防止记忆：模板化数值变体（±20% 随机化）让模型学到推理方法而非记住特定答案，公开数据集固定题干会导致过拟合。(4) 可控难度分层：生成数据有 easy/medium/hard 标注支持课程学习，公开数据集没有。(5) 系统提示词定制：8 个子领域各配专属 system prompt，公开数据集不具备。但项目不是永远不用公开数据——FinQA 等已在后续优化方向中，Phase 2 会批量导入并补充 chain-of-thought + system prompt + difficulty 标注后混入训练管线。
