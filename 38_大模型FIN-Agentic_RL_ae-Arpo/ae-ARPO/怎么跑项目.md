# ARAEPO 项目运行指南

## 能跑吗？

**能。** 但有几个前置条件和你需要手动修的东西。

### 跑之前必须确认的

| 条件 | 最低要求 | 推荐 |
|------|----------|------|
| GPU | 8×A100 40GB 或 8×H100 80GB | 8×H100 80GB |
| 显存（每卡） | 40GB（紧） | 80GB（舒适） |
| CUDA | 12.4+ | 12.4 |
| Python | 3.10 | 3.10 |
| 磁盘 | ~200GB（模型+checkpoint+数据） | 500GB+ |
| 网络 | 能访问 HuggingFace + BrightData API + wandb | - |

### 脚本里需要修的东西

打开 `ARAEPO/scripts/ARAEPO_Qwen3_14B_DeepResearch.sh`，以下占位符必须替换：

```
ACTOR_MODEL_PATH="<your_14B_model_path>"          # 改：你的 Qwen3-14B 路径
WANDB_API_KEY="<your_wandb_key>"                   # 改：你的 wandb key
SAVE_PATH="<your_checkpoint_save_dir>/rl/..."      # 改：checkpoint 保存目录

# 搜索工具配置（在 agent.yaml 里改）
api_key: [your_api_key]     # 改：BrightData API key
zone: [your_zone]            # 改：BrightData zone
conda_path: /path/to/conda   # 改：你的 conda 安装路径
```

## 一小时跑通的最小流程

别一上来就跑全量训练。先验证环境能 work。

### 1. 装环境（20分钟）

```bash
conda create -n araepo python=3.10 -y
conda activate araepo

# PyTorch
pip install torch==2.6.0 --index-url https://download.pytorch.org/whl/cu124

# Flash Attention（编译比较慢，耐心等）
pip install flash-attn --no-build-isolation

# verl 框架
cd ARAEPO/verl_araepo_entropy
pip install -e .

# 核心依赖
pip install ray[default] vllm wandb hydra-core tensordict<=0.6.2
```

### 2. 验证 verl 装好了（1分钟）

```bash
python -c "from verl.trainer.main_ppo import main; print('verl OK')"
python -c "from verl.workers.agent.tool_agent import ToolAgent; print('ToolAgent OK')"
python -c "import ray; ray.init(num_gpus=1); print('Ray + GPU OK'); ray.shutdown()"
```

### 3. 配搜索 API

编辑 `ARAEPO/verl_araepo_entropy/verl/workers/agent/agent.yaml`，把 `api_key` 和 `zone` 换成你的 BrightData 凭证。没有的话去 brightdata.com 注册一个，有免费额度。

### 4. 小规模试跑（30分钟）

拿7B模型 + 几条数据先跑通流程：

```bash
cd ARAEPO/scripts

# 把脚本里的关键参数改成小规模测试
# 如果不想直接改脚本，可以命令行 override：

python3 -m verl.trainer.main_ppo \
    --config-path=./config \
    --config-name=ppo_trainer_dr.yaml \
    algorithm.adv_estimator=grpo \

    actor_rollout_ref.model.path=<你的7B或14B模型路径> \
    actor_rollout_ref.actor.enable_entropy_balanced_clipping=True \
    actor_rollout_ref.actor.enable_entropy_balanced_advantage=True \
    actor_rollout_ref.actor.ppo_mini_batch_size=2 \
    actor_rollout_ref.actor.use_dynamic_bsz=True \
    actor_rollout_ref.actor.ppo_max_token_len_per_gpu=4096 \
    actor_rollout_ref.rollout.name=vllm \
    actor_rollout_ref.rollout.mode=sync_with_tool \
    actor_rollout_ref.rollout.gpu_memory_utilization=0.4 \
    actor_rollout_ref.rollout.n=4 \
    actor_rollout_ref.rollout.initial_rollouts=2 \
    actor_rollout_ref.rollout.beam_size=1 \
    actor_rollout_ref.rollout.tensor_model_parallel_size=1 \
    actor_rollout_ref.rollout.enable_dynamic_rollouts=False \
    actor_rollout_ref.rollout.multi_turn.enable=True \
    actor_rollout_ref.rollout.multi_turn.tool_config_path=../verl_araepo_entropy/verl/workers/agent/agent.yaml \
    reward_model.reward_manager=naive \

    trainer.val_before_train=False \

如果能跑完 1 个 epoch 不报错，环境就 OK 了。

```

### 7B 模型版本（更快、更省显存）

```bash

```

### 显存和时间估算

| 配置 | 每卡显存 | 总训练时间（1k数据，5 epoch） | 说明 |
|------|----------|------------------------------|------|
| Qwen3-14B, rollout_n=12, beam=2, seq=8K, 8×H100 | ~50-60GB | 4-6 小时 | 标准配置 |
| Qwen3-14B, rollout_n=4, beam=1, seq=4K, 8×A100 | ~35-40GB | 1-2 小时 | 快速实验 |
| Qwen2.5-7B, rollout_n=12, beam=2, seq=8K, 8×A100 | ~30-35GB | 2-4 小时 | 7B 版本 |
| Qwen2.5-7B, rollout_n=4, beam=1, seq=4K, 4×A100 | ~25-30GB | 1-2 小时 | 最省配置 |

**为什么 rollout 是时间瓶颈**：每步训练分两阶段——先生成（vLLM 推理，生成 64×12 条各 8K token 的序列），再更新（FSDP 梯度计算）。生成阶段占了 70-80% 的时间。

**显存去哪了**（14B 模型、8×H100 为例）：
- 模型权重（bf16，FSDP 分片）：~3.5GB/卡
- Adam 优化器状态：~7GB/卡
- 梯度：~3.5GB/卡
- 激活值（gradient checkpointing 开启）：~8-12GB/卡
- vLLM KV cache（gpu_memory_utilization=0.6）：~25-30GB
- 其他开销：~5GB

**降显存的方法**：
```bash
# 减小序列长度
data.max_response_length=2048       # 默认 6192

# 减小 batch / rollout
data.train_batch_size=32            # 默认 64
actor_rollout_ref.rollout.n=8       # 默认 12
actor_rollout_ref.rollout.initial_rollouts=4

# 开 offload
actor_rollout_ref.actor.fsdp_config.param_offload=True
actor_rollout_ref.actor.fsdp_config.optimizer_offload=True

# 降 vLLM 显存占用
actor_rollout_ref.rollout.gpu_memory_utilization=0.4

# 减小 tensor parallel
actor_rollout_ref.rollout.tensor_model_parallel_size=1

# 用 LoRA 替代全量微调
actor_rollout_ref.model.lora_rank=32
```

### GPU 数量选择

| GPU 数量 | 可行性 | 配置调整 |
|----------|--------|----------|
| 8 卡 | ✅ 标准配置 | 直接运行 |
| 4 卡 | ⚠️ 可以 | rollout_n 降到 8，initial_rollouts 降到 4 |
| 2 卡 | ⚠️ 勉强 | rollout_n 降到 4，initial_rollouts 降到 2，beam_size=1 |
| 1 卡 | ❌ 不行 | FSDP + vLLM + Ray 的开销会撑爆 |

## 完整三阶段流程

项目设计了三阶段流水线，但只跑 RL 训练也能用（跳过 SFT，直接用 instruct 模型作为起点）。

### 阶段一：SFT 冷启动（可选）

用 LLaMA-Factory 在 54K agentic 数据上做监督微调，给后续 RL 一个更好的初始化。

```bash
cd LLaMA-Factory
# 参考 arpo_train_sft/ 下的配置
pip install -e .
cd arpo_train_sft
bash sft_train.sh  # 需要先配好数据和模型路径
```

时间：8×A100 约 4-6 小时。显存：~40GB/卡。

### 阶段二：RL 训练

就是上面说的 `bash ARAEPO_Qwen3_14B_DeepResearch.sh`。

### 阶段三：评估

```bash
cd evaluation

# 先启动 vLLM 推理服务
bash vllm_scripts/launch_vllm.sh

# 跑评估
bash evaluate.sh
```

## 常见报错和解决

### ImportError: No module named 'verl'

`PYTHONPATH` 没设对。在 shell 脚本里确认这行：
```bash
export PYTHONPATH=${PARENT_DIR}/verl_araepo_entropy:$PYTHONPATH
```
或者手动 `pip install -e .` 在 `verl_araepo_entropy/` 目录下执行。

### vLLM OOM

降 `gpu_memory_utilization`：
```bash
actor_rollout_ref.rollout.gpu_memory_utilization=0.3
```

### Flash Attention 安装失败

编译需要 CUDA toolkit 和足够的内存。如果一直失败：
```bash
# 用预编译的 wheel
pip install flash-attn --no-build-isolation
# 或者跳过，改用 sdpa
export VLLM_ATTENTION_BACKEND=SDPA
```

### Ray 启动失败

```bash
ray stop --force  # 先停掉残留的 Ray 进程
ray start --head --num-gpus=8
```

### 工具调用一直失败

检查三件事：
1. BrightData API key 和 zone 是否有效（可以 curl 测试）
2. conda 路径是否正确（PythonTool 需要）
3. 网络是否能访问 api.brightdata.com

### 训练 loss NaN

大概率是 entropy_std 为 0 或 ratio 极端值。代码已经加了 `if entropy_std == 0: entropy_std = 1.0` 的兜底，但如果还 NaN：
- 降 `clip_ratio_high` 到 0.1
- 降 `entropy_weight` 到 0.1
- 检查你的模型 tokenizer 和数据集是否匹配

### 脚本里 ENABLE_MULTI_TURN 报错

这是个已知问题——shell 脚本引用了 `${ENABLE_MULTI_TURN}` 变量但忘了定义。在脚本里加上：
```bash
ENABLE_MULTI_TURN=True  # 或 False
```

## 数据格式

训练数据是 Parquet 文件，至少需要一个 `prompt` 列：
```
prompt: "请搜索2024年中国GDP增长率并给出分析"
reward_model.style: "rule"  # 可选
```

也可以用 `AEARPO/rl_datasets/` 下现有的数据直接跑。

## 监控

训练启动后看三个地方：
- **终端输出**：每一步的 reward、loss、kl 值
- **wandb**：完整指标曲线，包括我们新增的 Agent 指标（`branch/active_ratio`、`trajectory/loop_terminations` 等）
- **TensorBoard**：如果配了的话
