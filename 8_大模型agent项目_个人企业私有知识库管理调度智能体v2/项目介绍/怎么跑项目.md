# com_agent_chat — 怎么跑这个项目

## 环境配置

### 技术栈和依赖

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| Python | 3.12+ | 推荐3.12 |
| Docker + Docker Compose | 最新版 | 一键启动基础设施 |
| Neo4j | 5.28 Community | Docker内运行 |
| ChromaDB | 0.6.3 | Docker内运行 |
| Apache Kafka | 7.7.0 (Confluent) | Docker内运行（可选） |
| Tesseract OCR | 任意版本（中文包） | 图片OCR需要 |

### 环境变量

复制`.env.example`为`.env`并编辑：

```bash
cd code/python
cp .env.example .env
```

关键配置项：

```env
# ── LLM API（必填）──
OPENAI_API_KEY=你的API密钥          # 必填
OPENAI_BASE_URL=https://api.deepseek.com/v1  # 可用任意兼容OpenAI API的服务
OPENAI_MODEL=deepseek-chat          # DeepSeek / GPT-4o / Qwen 等

# ── Neo4j（使用Docker时保持不变）──
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=password

# ── 向量库（chroma 或 pgvector）──
VECTOR_STORE_TYPE=chroma            # 本地开发用chroma即可
CHROMA_HOST=localhost
CHROMA_PORT=8000

# ── API ──
API_HOST=0.0.0.0
API_PORT=8080
UPLOAD_DIR=./uploads

# ── Kafka（增量更新功能需要，可暂时跳过）──
KAFKA_BOOTSTRAP_SERVERS=localhost:9092

# ── 可选：跳过本地Embedding模型，只用API ──
# DISABLE_LOCAL_EMBEDDINGS=1
```

### LLM服务选择

项目使用OpenAI兼容API，可以接任意服务：

```env
# 选项1: DeepSeek（当前默认，性价比高）
OPENAI_BASE_URL=https://api.deepseek.com/v1
OPENAI_MODEL=deepseek-chat

# 选项2: OpenAI 官方
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o

# 选项3: 阿里通义千问
OPENAI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
OPENAI_MODEL=qwen-plus

# 选项4: 智谱GLM
OPENAI_BASE_URL=https://open.bigmodel.cn/api/paas/v4/
OPENAI_MODEL=glm-4

# 选项5: 本地Ollama（免费，无需API Key）
# 先执行: ollama pull qwen2
OPENAI_BASE_URL=http://localhost:11434/v1
OPENAI_API_KEY=ollama
OPENAI_MODEL=qwen2
```

## 运行步骤

### 方式一：Docker Compose 一键启动

```bash
# 1. 进入项目目录
cd code

# 2. 配置环境变量
cp python/.env.example python/.env
# 编辑 python/.env，填入OPENAI_API_KEY

# 3. 一键启动所有服务（Neo4j + ChromaDB + Kafka + API）
docker-compose up -d

# 4. 查看服务状态
docker-compose ps

# 5. 查看API日志
docker-compose logs -f api

# 6. 浏览器打开
# Web UI: http://localhost:8080
# API文档: http://localhost:8080/docs
# Neo4j Browser: http://localhost:7474
```

### 方式二：仅Docker启动基础设施，API本地运行（推荐调试用）

```bash
# 1. 只启动基础服务
cd code
docker-compose up -d neo4j chromadb

# 2. 安装Python依赖
cd python
pip install -r requirements.txt

# 3. 配置环境变量
cp .env.example .env
# 编辑.env

# 4. 启动API（开发模式，热重载）
python -m uvicorn api.main:app --host 0.0.0.0 --port 8080 --reload

# 5. 浏览器打开 http://localhost:8080
```

### 方式三：纯本地运行（无Docker）

如果不想装Docker，可以手动安装Neo4j和ChromaDB：

```bash
# 1. 安装Neo4j Community Edition
# 从 https://neo4j.com/download/ 下载安装
# 启动后设置密码为 password（或修改.env中NEO4J_PASSWORD）

# 2. ChromaDB不需要单独安装
# 项目使用嵌入式模式（PersistentClient），数据存在 chroma_data/ 目录

# 3. 安装Python依赖
cd code/python
pip install -r requirements.txt

# 4. 配置环境变量（Neo4j地址改为本地）
# NEO4J_URI=bolt://localhost:7687

# 5. 启动
python -m uvicorn api.main:app --reload
```

- `POST /api/ingest/batch` — 批量上传

- `POST /api/admin/update` — 手动触发更新

### Neo4j可视化查看知识图谱

```
浏览器打开: http://localhost:7474
用户名: neo4j
密码: password

# 查看所有实体和关系:
MATCH (n:Entity)-[r]->(m:Entity) RETURN n, r, m LIMIT 100

# 查看特定实体:
MATCH (e:Entity {name: "张三"})-[r]-(n) RETURN e, r, n

# 查看实体数:
MATCH (e:Entity) RETURN count(e)

# 删除所有数据:
MATCH (n) DETACH DELETE n
```

## 常见问题排查

### Q1: Docker启动后api容器退出

**症状**：`docker-compose ps` 显示 api 服务为 `Exit` 状态

**原因**：通常是没有配置API密钥或Neo4j还没完全启动

**解决方案**：
```bash
# 1. 确认.env文件存在且配置正确
ls -la code/python/.env

# 2. 先启动基础设施，等30秒再启动API
docker-compose up -d neo4j chromadb
sleep 30
docker-compose up -d api

# 3. 查看错误日志
docker-compose logs api
```

### Q2: Neo4j内存不足

**症状**：Neo4j容器反复重启，日志中有 `OutOfMemoryError`

**原因**：Neo4j默认需要较多内存

**解决方案**：
```bash
# Docker Desktop → Settings → Resources → Memory
# 建议分配至少 4GB

# 或在docker-compose.yml中限制Neo4j内存:
# neo4j服务添加:
#   environment:
#     NEO4J_server_memory_heap_initial__size: 512m
#     NEO4J_server_memory_heap_max__size: 1g
```

### Q3: Embedding模型下载慢或失败

**症状**：启动时长时间卡在 `[embedding_worker] Model loaded`

**原因**：`shibing624/text2vec-base-chinese` 模型需要从HuggingFace下载

**解决方案**：
```bash
# 方案1: 使用镜像站
export HF_ENDPOINT=https://hf-mirror.com

# 方案2: 手动预下载
python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('shibing624/text2vec-base-chinese')"

# 方案3: 跳过本地Embedding，只用API
# 在.env中设置:
DISABLE_LOCAL_EMBEDDINGS=1
```

### Q4: PDF上传后问答找不到内容

**症状**：上传了PDF，但问里面的内容时回答"知识库中暂无相关信息"

**原因**：可能PDF是扫描版（纯图片），文字提取失败且视觉兜底未触发

**解决方案**：
```bash
# 1. 检查入库响应:
#    chunks_count=0 → 解析失败
#    entities_count=0 → 抽取失败

# 2. 尝试用Markdown格式重新上传相同内容
# 3. 检查API日志中的错误信息
docker-compose logs api | grep -i error
```

### Q5: Windows下Tesseract OCR不可用

**症状**：`TesseractNotFoundError`

**解决方案**：
```bash
# 1. 安装Tesseract
# 下载: https://github.com/UB-Mannheim/tesseract/wiki
# 安装时勾选中文语言包

# 2. 或在Docker中运行（Dockerfile已内置Tesseract）
docker-compose up -d
```
